## P1 — real bugs, not launch-gating

7. **Publish flow ships blank modules.** `factory/scripts/publish.sh:57` + `factory/skills/publish.md:80` fetch `factory/templates/system-module.md` which does not exist on origin/main (renamed to `templates/module.md`); `curl -sf | sed >` writes an empty file that passes finalize()'s H1 grep. Every Author using /publish on a new module gets a blank scaffold. Fix: rename the two references.

8. **Invalid-key install half-dead-ends.** Rejected key: matrix row shows signup URL but closing block doesn't say "your key was rejected" plainly, and the bad key is persisted to `system/.api_key` before verification (`setup.sh:131-134`) so every bare re-run re-fails (`setup.sh:65-68`). Also: a malformed key not starting `alex_` is silently dropped by the arg parser (`setup.sh:37-40`) → silent keyless install; the "Invalid API key format" branch (`setup.sh:86-89`) is unreachable from args. Fix: don't persist unverified keys, plain rejection line in closing, catch non-`alex_` tokens.

9. **Lost key = stuck.** Once `installed_at` is stamped (first /call, `protocol.ts:361-376`), re-OAuth mints no new key (`routes.ts:476-479`) and no rotation endpoint exists. A member on a wiped machine who deleted the welcome email needs founder intervention (and admin account-remove purges their published content). Fix: self-serve rotate-key on re-OAuth (rotate = old hash out via existing `setAuthIndex`, already used at `routes.ts:509`).

10. **Half-install holes:** (a) `.block` counted in MISSING (`setup.sh:971`) but not in CORE_OK (`setup.sh:1086-1089`) — setup can print "read the block now" pointing at a file that didn't land, and payload's no-block branch then freestyles onboarding (`payload.sh:444`); (b) ctrl-C before `.setup_report` writes → next-session BLOCK fires, block.md step-0 reads a missing report — undefined; add "report missing = not ok" to block.md; (c) crash mid-onboarding: any >200-byte constitution file → `.block_complete` auto-touched (`payload.sh:434-436`) → half-onboarded lead looks "done" forever. Completion marker should be written by block completion, not inferred. (Block/payload/canon touches → `factory/ship.sh`.)

11. **verify-fetch.sh trust drift.** Header claims setup.sh installs it once (trust root); actually all four consumers lazy-fetch it unverified at first use (`install.md:23`, `publish.md:28/58/68`, `brief-setup.md:62`, `migrate.sh:4`). Fix: one `fetch_factory "scripts/verify-fetch.sh"` line in setup.sh.

12. **Rate-limit gaps:** `/auth/github`, `/auth/github/callback`, `/check-kin` unlimited; each `/auth/github` GET writes a KV record (write amplification), `/check-kin?code=<login>` is an unauthenticated membership oracle (`routes.ts:319-325`). Cheap: extend the existing 5/min/IP limiter (`worker.ts:296-374`).
